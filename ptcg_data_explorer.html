CARD DATA INSPECTOR

Every card is just structured text. This shows what the data is, what the pull script does to it, and how those fields become the moves your engine can choose between. Cards below are real, pulled live from the official data dump.

1. What the script does

ELI15: Cards have a tiny letter in the corner — the regulation mark. Standard format only allows certain letters. For the 2026 season that's H, I, J; everything G and older is retired. The script downloads all cards, throws away anything that isn't H/I/J, trims each card to just the parts a game needs, and saves one clean file.
# the whole legality test is two checks:
legal = (mark in {"H","I","J"}) and (legalities.standard == "Legal")

2. Two ways to get the data

# A) offline dump (no key, what the script uses)
raw.githubusercontent.com/PokemonTCG/pokemon-tcg-data/master/cards/en/sv9.json
# B) live API — query language is field:value
GET api.pokemontcg.io/v2/cards?q=regulationMark:H OR regulationMark:I OR regulationMark:J
Useful live filters: types:Fire · subtypes:ex · hp:[200 TO *] · name:"Dragapult ex". A free API key just raises how many you can pull per hour.

5. What move makes the next best move?

ELI15: The card data lists what a Pokémon can do. The best move depends on the game state — how much energy is attached, and how hurt the opponent is. The engine scores every legal action and picks the highest. Drag the sliders and watch the pick change. (This is a teaching toy, not the real engine — but the shape is exactly right.)
